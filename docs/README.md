# docs/

This directory holds your project's canonical documents — the source of truth that Claude Code references when making architectural decisions.

## What Goes Here

- **Architecture documents** — Tech stack, infrastructure, data model, deployment. The "how" of your project.
- **Seed documents / project briefs** — Vision, governance, organizational structure. The "what" and "why."
- **Onboarding guides** — How to set up the dev environment, deploy, access services.
- **Plans** (in `plans/`) — Execution plans generated by `/create-plan`. Auto-generated, timestamped.
- **Recipes** (in `recipes/`) — Reusable solutions captured by `/save-recipe` or auto-detected by `/execute`.

## Conventions

- Use semantic versioning in filenames: `architecture_v1_0.md`
- When updating, increment the version, update the date, add a changelog entry
- Reference these docs from `CLAUDE.md` so Claude Code knows they exist
- When renaming a versioned file, update all references in CLAUDE.md and other docs

## Getting Started

You don't need all of these on day one. Start with whatever context you have and add more as your project takes shape. Even a rough project brief is better than nothing — Claude Code uses whatever context you give it.
