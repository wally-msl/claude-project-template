# examples/

Minimal, realistic versions of the artifacts this template produces. The goal is to show what "good enough" looks like so you can start with something concrete rather than a blank file.

## What's here

- **`seed-docs/`** — A small project brief and a technical architecture doc. These are what Claude Code reads before doing anything. The examples are tiny on purpose — a half-page each is often enough to shape the first week of work.
- **`plans/`** — An execution plan in the format `/create-plan` produces. Status emojis, progress percentage, deploy checklist, no extra scope.
- **`recipes/`** — A recipe in the format `/save-recipe` and `/execute` produce. Problem, context, solution, why it works, gotchas.
- **`sred/`** — An SR&ED / R&D evidence log entry. Optional; only relevant if you're capturing research evidence.

## How to use

Copy anything here into your real `docs/`, `docs/plans/`, `docs/recipes/`, or `sred/` directory as a starting point. Then edit until it reflects your actual project. Don't keep the example text — it's a scaffold, not content.

## Why they are small

An example that's too polished teaches the wrong lesson. In real use, these artifacts are short, written quickly, and revised as you learn. The examples reflect that.
