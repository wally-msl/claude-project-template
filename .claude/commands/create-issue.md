I'm mid-development and thought of a bug, feature, or improvement. Capture it fast so I can keep working.

## Your Goal

Create a complete GitHub issue with:
- Clear title
- TL;DR of what this is about
- Current state vs expected outcome
- Affected components or services
- Relevant files that need touching
- Risk/notes if applicable
- Labels: type (bug/feature/improvement), priority, effort

## How to Get There

**Ask questions** to fill gaps — be concise, respect my time. I'm mid-flow. Usually need:
- What's the issue/feature
- Current behavior vs desired behavior
- Which component is affected

Keep questions brief. One message with 2-3 targeted questions beats multiple back-and-forths.

**Search for context** only when helpful:
- Grep codebase to find relevant files
- Note dependencies or risks

**Skip what's obvious** — if it's a straightforward bug, don't over-research. If type/priority is clear, don't ask.

**Create the issue** — write it to a file at `issues/ISSUE-[number]-[slug].md` or create it directly on GitHub if gh CLI is available.

**Keep it fast** — total exchange under 2 minutes.

## Labels

Type: `bug`, `feature`, `improvement`, `infrastructure`
Priority: `critical`, `high`, `normal`, `low`
